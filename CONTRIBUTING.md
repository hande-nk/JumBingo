# Contributing

## Branching

- `main` is always deployable. No direct commits.
- Branch naming: `feature/short-description`, `fix/short-description`, `chore/short-description`
- One branch per ticket/feature.

## Commits

Write commits in imperative mood, e.g. `Add login form validation`, not `Added` or `Adding`.

## Pull Requests

1. Push your branch, open a PR into `main`.
2. Fill out the PR template completely, don't skip sections.
3. Request review from at least one other dev.
4. Address review comments with new commits (don't force-push over review history until approved).
5. Squash and merge once approved.

## Code Review

Reviewers check for: does it work, is it readable, does it follow existing patterns, are there edge cases or error handling missed.

## Getting Unblocked

If you're stuck for more than ~30 minutes, post in the team channel rather than spinning. Include what you tried and the exact error.
